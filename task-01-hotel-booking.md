# Бизнес-функция: бронирование отеля

## 1. конкурентный анализ
| Сервис | Основные возможности |
|---|---|
| Яндекс Путешествия | Поиск отелей по месту, датам, количеству гостей, фильтрация, отзывы, бронирование онлайн |
| Авито | Поиск жилья по месту, датам, количеству гостей, фильтрация, отзывы, связь с владельцем |
| Суточно.ру | Поиск жилья по месту, датам, количеству гостей, фильтрация, отзывы, бронирование онлайн |

Основные функции:
- поиск жилья по месту, датам и количеству гостей;
- фильтрация результатов поиска;
- бронирование и оплата онлайн;

## 2. глоссарий
Пользователь: клиент сервиса, осуществляющий поиск и бронирование отеля.

Бронирование: процесс резервирования номера на определённый период времени.

Гость: лицо, для которого оформляется бронирование.

Сервис: информационная система, предоставляющая возможность поиска и бронирования отелей.

Фильтр: набор параметров для уточнения результатов поиска.

## 3. бизнес-цели
Целью бизнес-функции "бронирование отеля" является предоставление пользователям возможности самостоятельного поиска и бронирования номеров в отелях через сервис туроператора.

Для достижения данной цели необходимо:

- увеличить количество бронирований отелей онлайн;
- сократить время бронирования отеля пользователем;
- повысить удовлетворённость пользователей удобством пользования сервиса;
- снизить нагрузку на саппорт за счёт автоматизации бронирования;
- обеспечить своевременное уведомление пользователя о подтвержении бронирования.

## 4. описание бизнес-процесса
1. Пользователь переходит на главную страницу сервиса бронирования отелей.
2. Пользователь вводит город назначения, даты проживания и количество гостей.
3. Система выполняет поиск и отображает список доступных отелей.
4. Пользователь применяет фильтры для уточнения результатов поиска.
5. Пользователь выбирает подходящий отель и номер.
6. Система отображает подробную информацию о выбранном варианте размещения.
7. Пользователь подтверждает свой выбор и бронирует номер.
8. Пользователь оплачивает бронирование одним из доступных способов.
9. Система создаёт бронирование и отправляет пользователю уведомление об успешном бронировании.

## 5. user story
- как пользователь сервиса, я хочу найти отель по городу, датам проживания и количеству гостей, чтобы подобрать подходящий вариант размещения для своей поездки;
- как пользователь сервиса, я хочу использовать фильтры, чтобы быстрее найти отлеь, соответствующий моим требованиям по цене, рейтингу и условиям проживания;
- как пользователь сервиса, я хочу забронировать выбранный номер в отеле, чтобы гарантировать наличие места для проживания на время поездки;
- как пользователь сервиса, я хочу оплатить бронирование онлайн, чтобы завершить оформление заказа без обращения к саппорту;
- как пользователь сервиса, я хочу получить подтверждение бронирования после оплаты, чтобы не сомневаться в успешности оформления заказа.

## 6. ограничения
- доступно бронирование только тех отелей, которые размещены на сервисе туроператора;
- оплата доступна только поддерживаемыми сервисом способами;
- сервис должен учитывать наличие свободных мест на выбранные даты;
- поиск отелей доступен только по указанным туроператором направлениям;
- отмена бронирования возможна исключительно не менее чем за 7 календарных дней до даты заселения.

## 7. функциональные требования
1. Система должна предоставлять возможеость поиска отелей по месту.
2. Система должна позволять пользователю указывать даты заезда и выезда.
3. Система должна предоставлять возможность указания количества гостей.
4. Система должна предоставлять пользователю возможность фильтрации результатов поиска по различным параметрам (например стоимость, рейтинг, тип жилья и т.д.).
5. Система должна отображать подробную информацию об объекте размещения, включая описание, фотографии, стоимость проживания и отзывы.
6. Система должна предоставлять возможность оформления бронирования выбранного объекта размещения.
7. Система должна предоставлять возможность оплаты бронирования поддерживаемыми способами оплаты онлайн.
8. Система должна создавать бронирование после успешной оплаты и отправлять пользователю подтверждение.

## 8. нефункциональные требования
1. Время отображения результатов поиска не должно превышать 3 секунд при нормальной нагрузке.
2. Интерфейс должен быть интуинтивно понятным.
3. Сервис должен корректно функционировать как на ПК, так и на мобильных устройствах.
4. Передача персональных данных и платёжной информации должна быть защищённой.
5. Сервис должен быть доступен пользователям круглосуточно, за исключением периодов технического обслуживания.
6. Система должна обеспечивать сохранность данных о бронированиях и корректное восстановление работы после возникновения ошибок.

## 9. критерий готовности (DoR)
- определена бизнес-цель функции бронирования отелей;
- описан бизнес-процесс бронирования;
- сформирован глоссарий используемых терминов;
- подготовлены юзер стори;
- определены функциональные требования;
- определены нефункциональные требования;
- зафиксированы ограничения системы;
- выполнен анализ аналогичных решений конкурентов.

## 10. критерий завершённости (DoD)
- реализован поиск объектов размещения по месту назначения;
- реализован выбор дат заезда и выезда;
- реализована возможность указания количества гостей;
- реализована фильтрация результатов поиска;
- реализован просмотр карточки объекта размещения;
- реализована возможность оформления бронирования;
- реализована оплата бронирования онлайн;
- реализовано создание и отправка подтверждения бронирования;
- выполнено тестирование основных пользовательских сценариев;
